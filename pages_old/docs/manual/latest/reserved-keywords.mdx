# Reserved Keywords

> **Note**: Some of these words are reserved purely for compatibility with alternative Reason syntaxes, like OCaml, so they should not be used in your Reason source code.
>
> If you _need_ to use one of these names (for example, to interop with a JavaScript library that needs an object with a key of `"type"` like `{type: "elephant"}`), see the BuckleScript section for [handling JS naming collisions](/docs/reason-compiler/latest/handling-js-naming-collisions).

## A

- `and`
- `as`
- `assert`

## B

- `begin`

## C

- `class`
- `constraint`

## D

- `do`
- `done`
- `downto` (also see `to`)

## E

- `else` (used with `if`)
- `end`
- `esfun`
- `exception`
- `external`

## F

- `false` (a `bool`, along with `true`)
- `for`
- `fun`
- `function`
- `functor`

## I

- `if`
- `in`
- `include`
- `inherit`
- `initializer`

## L

- `lazy`
- `let`

## M

- `module`
- `mutable`

## N

- `new`
- `nonrec`

## O

- `object`
- `of`
- `open`
- `or`

## P

- `pri`
- `pub`

## R

- `rec`

## S

- `sig`
- `struct`
- `switch`

## T

- `then`
- `to`
- `true`
- `try`
- `type`

## V

- `val`
- `virtual`

## W

- `when`
- `while`
- `with`
